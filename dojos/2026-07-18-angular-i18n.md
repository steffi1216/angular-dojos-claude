# Angular Dojo: Internationalization (i18n)
**Datum:** 2026-07-18
**Dauer:** ~25 Minuten
**Level:** Fortgeschritten

## Lernziel
Du lernst, wie du eine Angular-Anwendung mit dem eingebauten i18n-System mehrsprachig machst – von der Markierung von Texten über die Extraktion der Übersetzungsdatei bis hin zur Darstellung lokalisierter Datums- und Zahlenformate.

## Hintergrund & Theorie

Angular bringt ein leistungsfähiges, kompilierzeit-basiertes Internationalisierungssystem mit. Im Gegensatz zu Laufzeit-i18n-Bibliotheken (z. B. ngx-translate) werden Übersetzungen bei Angular i18n direkt in den Build-Prozess eingebettet – jede Sprache bekommt ihr eigenes Build-Artefakt, was maximale Performance garantiert.

Wichtige Konzepte:

- **`i18n`-Attribut**: Markiert Template-Texte zur Übersetzung. Wird auf HTML-Elemente oder als `i18n-<attr>` für Attribute angewendet.
- **`$localize`**: Template-Tag für Strings in TypeScript-Code außerhalb von Templates.
- **`ng extract-i18n`**: CLI-Befehl, der alle markierten Texte in eine `messages.xlf`-Datei extrahiert.
- **ICU-Ausdrücke**: Ermöglichen pluralisierungs- und geschlechtsabhängige Übersetzungen direkt im Template.
- **`LOCALE_ID`-Token**: Ermöglicht die Injektion der aktiven Locale in Services und Komponenten.
- **Locale-abhängige Pipes**: `DatePipe`, `CurrencyPipe`, `DecimalPipe` und `PercentPipe` passen ihre Ausgabe automatisch an die aktive Locale an.

Jede Sprache wird beim Build als separates Bundle erzeugt (`ng build --localize`). Im Dev-Modus kann eine Locale mit `--configuration=de` aktiviert werden.

## Aufgabe

Baue eine kleine „Event-Anzeige"-Komponente, die:
1. Alle dargestellten Texte mit `i18n`-Attributen markiert
2. Einen ICU-Plural-Ausdruck für Teilnehmerzahlen verwendet
3. Datum und Preis über Locale-bewusste Pipes formatiert
4. Einen `$localize`-String in einem Service nutzt
5. Eine deutsche Übersetzungsdatei (`messages.de.xlf`) mit übersetzten Texten enthält

### Schritte

1. **Projekt vorbereiten**: Füge `de` als Locale zu `angular.json` hinzu und registriere `@angular/common/locales/de` in `app.config.ts`.
2. **`EventCardComponent` erstellen**: Zeige Titel, Datum, Preis und Teilnehmerzahl an. Markiere alle statischen Texte mit dem `i18n`-Attribut.
3. **ICU-Ausdruck einbauen**: Nutze eine `{count, plural, =0 {...} =1 {...} other {...}}`-Syntax für die Teilnehmeranzahl.
4. **Service mit `$localize`**: Erstelle einen `EventService`, der einen Bestätigungs-String via `$localize` zurückgibt.
5. **Übersetzungen extrahieren & übersetzen**: Führe `ng extract-i18n` aus und lege `messages.de.xlf` mit deutschen Übersetzungen an.
6. **Build testen**: Starte die App mit `ng serve --configuration=de` und prüfe, ob Datum und Preis im deutschen Format erscheinen.

## Hints

<details>
<summary>Hint 1 – i18n-Attribut und benutzerdefinierte ID</summary>

Du kannst einer Übersetzung eine stabile ID und eine Beschreibung geben, damit Übersetzer Kontext haben:

```html
<h2 i18n="Event title|The title of an event card@@eventCard.title">
  Upcoming Event
</h2>
```

Das `@@eventCard.title` ist die stabile ID, die in der XLF-Datei erscheint. Ohne sie generiert Angular eine hash-basierte ID, die sich bei Textänderungen verändert.

</details>

<details>
<summary>Hint 2 – ICU-Plural-Ausdruck</summary>

ICU-Ausdrücke werden direkt im Template in geschweiften Klammern geschrieben:

```html
<p i18n>
  {attendeeCount, plural,
    =0 {No attendees yet}
    =1 {One attendee}
    other {{{attendeeCount}} attendees}
  }
</p>
```

Beachte: Die drei geschweiften Klammern `{{{attendeeCount}}}` sind korrekt – die äußeren gehören zur ICU-Interpolation, die mittleren zur Angular-Interpolation.

</details>

<details>
<summary>Hint 3 – Locale in angular.json registrieren</summary>

In `angular.json` unter `projects > <name> > i18n`:

```json
"i18n": {
  "sourceLocale": "en-US",
  "locales": {
    "de": {
      "translation": "src/locale/messages.de.xlf"
    }
  }
}
```

Und im `build`-Target `"localize": ["de"]` setzen, oder für `serve` eine Konfiguration `"de"` anlegen:

```json
"configurations": {
  "de": {
    "localize": ["de"]
  }
}
```

</details>

<details>
<summary>Hint 4 – $localize in TypeScript</summary>

```typescript
import '@angular/localize/init'; // einmalig in main.ts importieren

@Injectable({ providedIn: 'root' })
export class EventService {
  getConfirmationMessage(title: string): string {
    return $localize`You have successfully registered for ${title}:eventTitle:.`;
  }
}
```

Der `:eventTitle:`-Marker nach der Interpolation ist ein optionaler Platzhalter-Name, der in der XLF-Datei sichtbar ist und Übersetzer/innen hilft.

</details>

## Beispiellösung

```typescript
// app.config.ts
import { ApplicationConfig } from '@angular/core';
import { provideRouter } from '@angular/router';
import { registerLocaleData } from '@angular/common';
import localeDe from '@angular/common/locales/de';
import { LOCALE_ID } from '@angular/core';

registerLocaleData(localeDe);

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter([]),
    { provide: LOCALE_ID, useValue: 'de' },
  ],
};
```

```typescript
// event-card.component.ts
import { Component, input } from '@angular/core';
import { DatePipe, CurrencyPipe } from '@angular/common';

export interface Event {
  title: string;
  date: Date;
  price: number;
  attendeeCount: number;
}

@Component({
  selector: 'app-event-card',
  standalone: true,
  imports: [DatePipe, CurrencyPipe],
  template: `
    <div class="event-card">
      <h2 i18n="Event title@@eventCard.title">Upcoming Event</h2>
      <h3>{{ event().title }}</h3>

      <p>
        <span i18n="Date label@@eventCard.dateLabel">Date:</span>
        {{ event().date | date:'fullDate' }}
      </p>

      <p>
        <span i18n="Price label@@eventCard.priceLabel">Price:</span>
        {{ event().price | currency:'EUR' }}
      </p>

      <p i18n="Attendee count@@eventCard.attendees">
        {event().attendeeCount, plural,
          =0 {No attendees yet}
          =1 {One attendee}
          other {{{event().attendeeCount}} attendees}
        }
      </p>

      <button (click)="register()" i18n="Register button@@eventCard.register">
        Register now
      </button>
    </div>
  `,
})
export class EventCardComponent {
  event = input.required<Event>();

  register(): void {
    console.log($localize`Registration confirmed for ${this.event().title}:eventTitle:.`);
  }
}
```

```xml
<!-- src/locale/messages.de.xlf -->
<?xml version="1.0" encoding="UTF-8" ?>
<xliff version="1.2" xmlns="urn:oasis:names:tc:xliff:document:1.2">
  <file source-language="en-US" target-language="de" datatype="plaintext">
    <body>
      <trans-unit id="eventCard.title" datatype="html">
        <source>Upcoming Event</source>
        <target>Kommendes Event</target>
      </trans-unit>
      <trans-unit id="eventCard.dateLabel" datatype="html">
        <source>Date:</source>
        <target>Datum:</target>
      </trans-unit>
      <trans-unit id="eventCard.priceLabel" datatype="html">
        <source>Price:</source>
        <target>Preis:</target>
      </trans-unit>
      <trans-unit id="eventCard.attendees" datatype="html">
        <source>{VAR_PLURAL, plural, =0 {No attendees yet} =1 {One attendee} other {<x id="INTERPOLATION"/> attendees}}</source>
        <target>{VAR_PLURAL, plural, =0 {Noch keine Teilnehmer} =1 {Ein Teilnehmer} other {<x id="INTERPOLATION"/> Teilnehmer}}</target>
      </trans-unit>
      <trans-unit id="eventCard.register" datatype="html">
        <source>Register now</source>
        <target>Jetzt anmelden</target>
      </trans-unit>
    </body>
  </file>
</xliff>
```

## Weiterführendes

- **Kompilierzeit vs. Laufzeit**: Für dynamisch ladbare Übersetzungen (z. B. per HTTP) ist [ngx-translate](https://github.com/ngx-translate/core) oder `@ngx-i18n/core` eine Alternative – aber zu Lasten von Tree-Shaking und Build-Optimierungen.
- **Pluralisierung mit `MessageFormat`**: Angular unterstützt auch den `MessageFormat`-Standard mit der `@angular/localize`-Option `--format=json` für komplexere Fälle.
- **Offizielle Docs**: [angular.dev/guide/i18n](https://angular.dev/guide/i18n) – inklusive Build-Konfiguration für mehrsprachige Deployments auf einem CDN.
- **Extraktion automatisieren**: Integriere `ng extract-i18n --output-path src/locale` in deine CI-Pipeline, um sicherzustellen, dass neue Texte nie unbemerkt unübersetzt bleiben.
